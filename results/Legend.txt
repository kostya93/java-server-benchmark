Линии на графиках (архитектуры):
1 - TCP. Сервер обрабатывает запросы в асинхронном режиме.
2 - TCP. Клиент устанавливает постоянное соединение. Сервер использует кеширующий пул потоков для общения с конкретными клиентами. Задача для этого пула потоков - общение с клиентом, пока не разорвется соединение.
3 - TCP. Клиент устанавливает постоянное соединение. Сервер производит неблокирующую обработку. Каждый запрос обрабатывается пуле потоков фиксированного размера. Сервер работает с сокетами в однопоточном режиме.
4 - TCP. Каждый запрос клиента - новое установление TCP соединения. Сервер - однопоточный с последовательной обработкой клиентов и закрытием соединения после отсылки ответа
5 - TCP. Клиент устанавливает постоянное соединение. Сервер создает отдельный поток на общение с конкретным клиентом.
6 - UDP. Создается отдельный поток на обработку каждого запроса
7 - UDP. Используется фиксированный пул потоков для обработки запросов

Графики:
ClientByDelta - Среднее время работы клиента в зависимости от delta, ms.
ClientByNumOfClients - Среднее время работы клиента в зависимости от количества клиентов, ms.
ClientByNumOfElements - Среднее время работы клиента в зависимости от количества элементов в массиве, ms.

ClientServerByDelta - Время обработки клиента на сервере в зависимости от delta, ms
ClientServerByNumOfClients - Время обработки клиента на сервере в зависимости от количества клиентов, ms
ClientServerByNumOfElements - Время обработки клиента на сервере в зависимости от количества элементов в массиве, ms

RequestServerByDelta - Время обработки запроса на сервере в зависимости от delta, ms
RequestServerByNumOfClients - Время обработки запроса на сервере в зависимости от количества клиентов, ms
RequestServerByNumOfElements - Время обработки запроса на сервере в зависимости от количества элементов в массиве, ms
